## 商品特色分析

### 為什麼選擇清心福全珍珠奶茶？

#### 1. **經濟指標性強**
- 清心福全是大學生日常消費的代表商品
- 價格變動直接反映通貨膨脹對年輕族群的影響
- 自2025年以來已多次調整價格

#### 2. **數據可取得性高**
- 全台連鎖店面，價格統一
- 價格變動通常有新聞報導
- 容易建立完整的歷史價格記錄

#### 3. **符合作業主題**
- 屬於「民生消費品」範疇
- 能有效說明「體感通膨」概念
- 易於對比官方CPI與個人感受

#### 4. **學生相關性**
- 大學生高度相關的日常消費
- 更能引起目標用戶的共鳴
- 便於蒐集同學的價格數據

## 資料模型

### 價格表 (Prices Table)

```
┌──────────────────────────────────────┐
│           prices                     │
├──────────────────────────────────────┤
│ id          | INTEGER (PK)           │
│ date        | TEXT (YYYY-MM-DD)      │
│ productName | TEXT                   │
│ price       | REAL (TWD)             │
│ createdAt   | DATETIME               │
└──────────────────────────────────────┘
```

#### 資料型別說明
- **id**: 自增主鍵，唯一識別每筆記錄
- **date**: 記錄的日期，格式為 YYYY-MM-DD
- **productName**: 商品名稱，固定為「清心福全珍珠奶茶」
- **price**: 商品價格，精確到小數點後兩位
- **createdAt**: 記錄建立時間，自動為當前時間

## API 規格

### 1. 新增價格記錄

**端點**: `POST /api/prices`

**請求格式**:
```json
{
  "date": "2026-05-12",
  "productName": "清心福全珍珠奶茶",
  "price": 55.00
}
```

**成功回應 (201)**:
```json
{
  "id": 1,
  "date": "2026-05-12",
  "productName": "清心福全珍珠奶茶",
  "price": 55.00
}
```

**錯誤回應 (400)**:
```json
{
  "error": "缺少必要欄位"
}
```

---

### 2. 取得所有記錄

**端點**: `GET /api/prices`

**成功回應 (200)**:
```json
[
  {
    "id": 1,
    "date": "2026-05-12",
    "productName": "清心福全珍珠奶茶",
    "price": 55.00,
    "createdAt": "2026-05-12 10:00:00"
  },
  {
    "id": 2,
    "date": "2026-05-11",
    "productName": "清心福全珍珠奶茶",
    "price": 50.00,
    "createdAt": "2026-05-11 15:30:00"
  }
]
```

---

### 3. 依日期搜尋

**端點**: `GET /api/prices/search?date=2026-05-12`

**查詢參數**:
| 參數 | 型別 | 說明 |
|------|------|------|
| date | string | 搜尋日期 (YYYY-MM-DD) |

**成功回應 (200)**:
```json
[
  {
    "id": 1,
    "date": "2026-05-12",
    "productName": "清心福全珍珠奶茶",
    "price": 55.00
  }
]
```

---

### 4. 刪除記錄

**端點**: `DELETE /api/prices/:id`

**路徑參數**:
| 參數 | 型別 | 說明 |
|------|------|------|
| id | integer | 記錄 ID |

**成功回應 (200)**:
```json
{
  "success": true
}
```

---

### 5. 取得統計資訊

**端點**: `GET /api/statistics`

**成功回應 (200)**:
```json
{
  "totalRecords": 10,
  "minPrice": 45.00,
  "maxPrice": 60.00,
  "avgPrice": 52.50,
  "distinctDates": 8
}
```

## 使用流程

### 典型使用場景

```
1. 使用者訪問應用程式
   ↓
2. 系統載入所有歷史記錄與統計資訊
   ↓
3. 使用者選擇日期、輸入價格
   ↓

   ↓
5. 後端驗證並存儲到資料庫
   ↓
6. 前端即時更新表格與統計資訊
   ↓
7. 使用者可搜尋特定日期或刪除記錄
```
